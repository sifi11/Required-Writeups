# Forensics Git 2

## How to solve:
I downloaded the disk image and opened it with Autopsy to get a GUI to make it easier to navigate. I then went to the fourth volume because that was the only one with actual files in it that were useful. I then went to `\home\ctf-player\Code` and found the `killer-chat-app` folder which I extracted using Autopsy. I then went to the directory where I extracted it and ran `git log`, but no flag was there and no commits were there either, so I looked around a little bit and couldn't find anything. I looked back at the challenge and saw something about someone interrupting deletion. So I asked Claude how I would access Git files that weren't entirely deleted, and he said to use `git fsck --unreachable` to see what git blobs or trees weren't reachable. I then ran `git fsck --lost-found` to see which ones might be useful. I saw a dangling blob that looked suspicious, so I ran `git log --all <commit hash>` and saw a commit about adding a secret hideout log. So I ran `git show <commit hash>` and saw someone telling someone to use a password to get into a secret hideout, that "password" was really just the flag.
